A diferença entre Hash e criptografia:

Bem antes de fazermos uma análise, é preciso entender o que é ambos:

Hash: O valor fixo gerado a partir de dados usandos uma função matemática que serve para identificar ou verificar informações de forma rápida e segura então resumidamente... É o resumo de um dado.

Criptografia: É o processo de transformar informações em um formato ilegível para quem não possui a chave.

Certo e qual a diferença?:

A diferença é que o hash diferente da criptografia ele identifica e verifica dados, enquanto a criptografia protege dados para que só os individuos que tenham a chave acessem as informações, outra diferença é a reversibilidade que bem.. no hash isso não existe, uma vez que não dá pra voltar ao dado original, enquanto a criptografia dá pra decifrar se tiver a chave correta.

Onde é usado cada um?:

O hash é muito utilizado em senhas armazenadas e verificação de arquivos, já a criptografia é usada em mensagens privadas e transações bancárias (Isso normalmente, não que ele não seja usado para outras coisas como criptografar um HD).

Como identificar:

Hash:

1) Sempre possui tamanho fixo, mesmo se o dado original for enorme ou minúsculo.

2) Não é possível voltar ao dado original.

3) O Hash é normalmente usado em verificação de integridade, autenticar senhas e gerar assinaturas digitais.

Criptografia:

1) O tamanho do dado criptografado geralmende depende do tamanho do dado original.

2) É possível descriptografar se você tiver a chave.

3) Normalmente usado para proteger comunicações (Exemplo: Email), armazenar dados sensíveis e enviar mensagens privadas.

Exemplos de Hash e criptografia:

Hash:

SHA-256, bcrypt, Argon2, MD5, SHA-1, assinaturas digitais e identificadores de blocos em blockchain.

Criptografia:

end-to-end encryption (usada no Whatsapp ou signal), dados de cartões de crédito ou bancários armazenados em bancos, VPN e conexões HTTPS (Exemplo: TLS/SSL), arquivos protegidos por senha (Exemplo: AES, RSA e ChaCha20).

Dica fácil para identificar:

Se não der para recuperar o original --> Hash

Se dá para recuperar com a chave correta --> Criptografia.
